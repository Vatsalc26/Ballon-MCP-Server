# MCP Listings

This page keeps the Balloon MCP listing strategy honest and reproducible.

## Recommended Order

1. Official MCP Registry
2. Anthropic Connectors Directory
3. Smithery
4. Glama
5. 1-2 curated community lists

## Why This Order

### Official MCP Registry

This is the best first listing target because it is the official MCP registry surface.

Use it first for legitimacy and discoverability.

Important nuance:

1. the official registry is the strongest legitimacy layer
2. but actual user discovery may still happen through downstream marketplaces and directories that consume registry metadata

### Anthropic Connectors Directory

This is a high-visibility surface for Claude users, but it is curated.

Treat it as valuable and reputable, but not guaranteed.

### Smithery

This is a real third-party discovery and publishing surface.

It is worth listing there after the official registry, especially because users can discover and connect to servers there from MCP clients.

### Glama

This is also a real third-party directory and worth listing after the official and curated surfaces.

It is especially useful for browse-style discovery because it has a large searchable MCP directory.

### Community lists

Use curated GitHub lists for extra discovery and contributor visibility, not as the core source of legitimacy.

## Before Listing Anywhere

Make sure you have:

1. a working public repo
2. install docs
3. security policy
4. contribution guide
5. an honest readiness statement
6. at least one reproducible demo or benchmark path

## Current Balloon Readiness Reading

Balloon is appropriate to list as:

1. a public alpha
2. a working external approximation of the Balloon architecture
3. an anti-drift MCP server for long AI sessions

Do not list it as:

1. production-ready for all hosts
2. hidden-state access for closed models
3. already proven winner of SlopCodeBench

## Suggested Listing Metadata

Keep the short description close to:

`Balloon MCP is an anti-drift MCP server for long AI sessions. It builds a structured profile, audits drift, retrieves relevant anchors, and applies bounded corrective pressure through deterministic, assist, and staged external lanes.`
